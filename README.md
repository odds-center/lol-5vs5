# LoL 5vs5 내전 | 관악구 피바라기

MMR 기준 5:5 내전 팀 분배 및 역할(탑/정글/미드/원딜/서폿) 랜덤 배정 웹 앱입니다.  
모든 데이터는 **localStorage**에만 저장되며, 서버 없이 Vercel 등에 정적 배포할 수 있습니다.

- **목적**: 롤 내전 시 MMR 기반 5:5 팀 분배 및 역할 배정
- **스택**: Next.js 14 (App Router), React, TypeScript, Tailwind CSS
- **배포**: Vercel 정적 배포, 클라이언트 전용(API/DB 미사용)

## 기능 요약

| 기능 | 설명 |
|------|------|
| **참가자 10명 입력** | 닉네임, MMR, 역할군(선호)을 10개 고정 슬롯에 입력. 역할군은 라인당 최대 2명 선택 가능. |
| **팀 나누기** | 참가자 탭에서 10명을 채운 뒤 실행. MMR 스네이크 드래프트로 1팀·2팀 분배 후 결과 탭으로 이동. |
| **다시 나누기** | 참가자 명단(10명)을 **랜덤 셔플**해 1팀·2팀을 다시 구성. 누를 때마다 다른 조합. |
| **역할 랜덤 배정** | 팀이 정해진 뒤, 각 팀 내에서 5명을 탑/정글/미드/원딜/서폿에 1:1 랜덤 배정. |
| **참가자 수정** | 결과 탭에서 참가자 탭으로 이동해 명단 수정 가능. |
| **역할 아이콘** | 탑·정글·미드·원딜·서폿 라인 아이콘 표시 (`public/roles/*.svg`). |
| **데이터 유지** | 참가자 명단·마지막 팀/역할 결과는 localStorage에 저장되어 새로고침 후에도 유지. |

## 화면 구성

- **타이틀**: LoL 5vs5 내전 (부제: 관악구 피바라기)
- **탭**: **참가자** | **결과** — 탭 전환으로 한 화면에서 입력/결과 구분
- **참가자 탭**
  - 테이블: `#` | 닉네임 | MMR | 역할군 (라인당 최대 2명)
  - 10행 고정 입력란 + 역할 select(미정/탑/정글/미드/원딜/서폿)
  - **팀 나누기** 버튼 (10명일 때만 활성)
- **결과 탭**
  - 팀 A / 팀 B 카드 (MMR 합계·평균, 역할별 명단 + 아이콘)
  - **다시 나누기** · **역할 랜덤 배정** · **참가자 수정** 버튼

---

## 사용 방법

1. **참가자** 탭에서 10명의 닉네임·MMR을 입력하고, 필요 시 역할군(선호)을 선택한다.
2. **팀 나누기**를 누르면 MMR 기준으로 1팀·2팀이 나뉘고 **결과** 탭으로 이동한다.
3. **역할 랜덤 배정**을 누르면 팀 내에서만 역할이 랜덤으로 다시 배정된다.
4. **다시 나누기**를 누르면 같은 10명을 랜덤으로 섞어 1팀·2팀이 새로 구성된다(누를 때마다 조합 변경).
5. 명단을 바꾸려면 **참가자 수정**으로 참가자 탭으로 돌아가 수정한 뒤, **팀 나누기**를 다시 실행하면 된다.

## 데이터 저장 (localStorage)

- **키**
  - `lol-5vs5-players`: 참가자 10슬롯 배열 (닉네임, MMR, 역할군)
  - `lol-5vs5-last-assignment`: 마지막 팀·역할 배정 결과 (팀 A/B, 역할별 배정, 생성 시각)
- **데이터 구조**
  - 참가자: `id`(slot-0~slot-9), `name`, `mmr`, `rolePreference`(탑/정글/미드/원딜/서폿 또는 미정)
  - 배정 결과: 팀 A/B 각 5명, 팀별 역할 5개(탑·정글·미드·원딜·서폿)에 1:1 매핑

---

## 동작 방식 상세

| 동작 | 설명 |
|------|------|
| **팀 나누기** | 10명을 MMR 내림차순 정렬 후 **스네이크 드래프트**(1·2위→각 팀, 3·4위→각 팀 …)로 1팀·2팀 분배. 동일 입력이면 항상 같은 조합. |
| **다시 나누기** | 참가자 10명을 **랜덤 셔플**한 뒤 앞 5명=1팀, 뒤 5명=2팀. 누를 때마다 다른 조합. |
| **역할 랜덤 배정** | 팀은 그대로 두고, 각 팀 내 5명을 5역할에 **1:1 랜덤** 배정. 선호 역할은 사용하지 않음. |

## 실행 및 배포

### 로컬 실행

```bash
npm install
npm run dev
```

브라우저에서 [http://localhost:3000](http://localhost:3000) 접속.

### 빌드

```bash
npm run build
```

### 배포 (Vercel)

- 저장소 연결 후 Vercel에서 자동 빌드·배포.
- 환경 변수 없이 사용 가능 (클라이언트 전용).
